Authentication Troubleshooting Guide

Common Authentication Issues

If you're getting "INVALID_LOGIN" errors, here are the most common causes and solutions:

Security Token Issues (Most Common)
⚠️ Even with a fresh security token, you might still have issues if:
  • The token wasn't copy-pasted correctly (extra spaces, etc.)
  • You're using an old token from a previous email
  • The token is being entered in the wrong format
🔧 Solutions to try:
  1. Append token to password method:
    • If your password is: mypassword123
    • And your security token is: ABCDEF1234567890
    • Enter as password: mypassword123ABCDEF1234567890
    • Leave the security token field empty
  2. Double-check your latest token:
    • Go to Setup → My Personal Information → Reset Security Token
    • Click "Reset Security Token" again (even if you just did it)
    • Use the NEWEST token from your email
  3. Copy-paste carefully:
    • Copy the token from your email
    • Paste into a text editor first to check for extra spaces
    • Then copy from the text editor to the login form
IP Address Restrictions

Your Salesforce organization might restrict API access to specific IP addresses.

🔧 Solutions:
  1. Contact your Salesforce administrator and ask them to:
    • Whitelist your current IP address for API access
    • Or temporarily disable IP restrictions for testing
  2. Check your current IP: Your IP is likely changing if you're on a mobile connection
  3. Use OAuth instead: OAuth authentication often bypasses IP restrictions
User Profile & Permissions

Your user profile might not have API access enabled.

🔧 What to check:
  1. API Enabled permission:
    • Contact your admin to verify "API Enabled" is checked on your user profile
    • This is found in Setup → Users → [Your User] → Edit
  2. Profile permissions:
    • Your user profile needs "API Enabled" permission
    • Some profiles have API access disabled by default
Account Status Issues
🔧 Quick checks:
  1. Test web login first:
    • Can you log into Salesforce normally through the web interface?
    • If not, your account might be locked or frozen
  2. Check for account locks:
    • Too many failed login attempts can temporarily lock your account
    • Wait 15-30 minutes before trying again
Environment & Configuration
🔧 Double-check these settings:
  1. Sandbox vs Production:
    • Make sure you're connecting to the right environment
    • Sandbox credentials won't work for Production and vice versa
  2. Username format:
    • Use your full email address as the username
    • For sandboxes, it might be: username@company.com.sandboxname
Quick Test Recommendation

Try this right now:

  1. Reset your security token one more time
  2. Copy the new token from your email
  3. Go back to the login page
  4. Enter your username and password
  5. Leave the security token field EMPTY
  6. Instead, append the token directly to your password
  7. Click "Login with SOAP API"
