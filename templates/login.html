OAuth 2.0 Authentication

Connect using OAuth 2.0 to securely access your Salesforce org. This is the recommended method for most users.

When to use OAuth:
  • Recommended for most users
  • Works with all Salesforce editions
  • No need for security tokens
Permissions Required:
  • Access basic information (your user identity)
  • Access and manage your data (to read schema and insert test data)
  • Perform requests on your behalf at any time

Direct SOAP API Login
Manage Saved

Connect directly using your Salesforce username and password via the SOAP API. This is an alternative if OAuth isn't working for your organization.

Common Issues & Solutions:
  • Invalid Login Error: Most often caused by missing security token
  • Security Token: Required for API access from unknown IPs
  • IP Restrictions: Check if your org has IP restrictions enabled
  • Account Status: Verify your account is active and not locked
Detailed Help Get Security Token
Required for API access from unrecognized IPs. You can either:
  1. Enter it in this field, OR
  2. Append it to your password (e.g., if password is "mypass" and token is "ABCDE", enter "mypassABCDE" as your password)
💡 How to get your security token:
  1. Log into Salesforce in your browser
  2. Go to Setup → My Personal Information → Reset Security Token
  3. Click "Reset Security Token" - it will be emailed to you
  4. Use that token here or append it to your password

Environment Setup

This application requires proper configuration of Salesforce Connected App credentials. Make sure the following environment variables are set:

  • SALESFORCE_CLIENT_ID - Your Connected App's Consumer Key
  • SALESFORCE_CLIENT_SECRET - Your Connected App's Consumer Secret
  • SALESFORCE_REDIRECT_URI - The callback URL for OAuth
  • OPENAI_API_KEY - Your OpenAI API Key for ChatGPT
